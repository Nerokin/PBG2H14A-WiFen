Modellen:
- Einzelne Java Klasse
- Enthalten ein gekapseltes Datenmodell (Möglichst keinerlei Beziehungen nach außen)

Services:
- Stellen Funktionalität bereit
- Definieren Schnittstellen
- Können eine Implementierung (Unterorder/paket 'impl', muss nicht vollständig sein) vorgeben.

Oberflächen-Komponenten:
- Haben keine weitere interne Struktur (Erweitern eine bestimmte JavaFX-Oberflächenkomponente)

Oberflächen-Strukturen:
- Sind eine komplexe Variante von Oberflächen-Komponenten
- Haben eine interne Struktur (Erweitern die JavaFX-Layout Klasse)
- Bestehen aus drei Dateien ({Strukturelement}.java, {Strukturelement}.css, {Strukturelement}.fxml), mit entsprechender Ordnerstruktur
- Definieren normalerweise keine Schnittstelle

Beispiel:

wifen.client {
	Main.java
	
	wifen.client.ui {
	
		<controllers>
			*.java extends Layout
		</controllers>
		
		wifen.client.ui.views {
			*View.fxml;
		}
		
		wifen.client.ui.css {
			*Style.css
		}
		
		wifen.client.ui.components {
			?.java extends Node
		}
	}
}

wifen.server {
	Main.java
}

wifen.commons {

	wifen.commons.models {
	}
	
	wifen.commons.network {
		wifen.commons.network.impl {
		}
	}
	
}
